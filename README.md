# APost Script Runner README

This extension is script Runner for APost.

## Features

### Input Initialize code

Before scripting, you need to add initialize code on top in script file.

The Input Initialize Code command can do this automatically.

![Input Initialize code](/images/Description_InputInitializeCodeline.png)

Initialize.csx has reference for major APost interface files.

> Tip: You can find the Initialize file as follow. %InstallationFolder%\ScriptEditor\VSCODE\Initialize.csx

### Run script file to APost

When you finish script work, you can run the script file to APost.

The Run Script file to APost command let the script file run to APost.

![Run script file to APost](/images/Description_RunScriptFileToAPost.png)

## How to using APost API

APost provide API reference document and serveral useful examples.

### API document

`%InstallationFolder%\Help\APostScriptHelp.chm`

### Examples

`%InstallationFolder%\ScriptEditor\VSCODE\Examples`

## Requirements

The APost application should be installed.

## Extension Settings

This extension contributes the following settings:

* `APostScriptRunner.APostPath`: set APost installation folder. eg) "C:/Program Files/FunctionBay, Inc/Standalone Post V0R3"

## Known Issues

This extension does not support to debug.
